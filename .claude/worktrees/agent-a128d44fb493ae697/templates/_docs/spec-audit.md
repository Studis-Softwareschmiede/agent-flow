# Spec-Audit-Log

> Logbuch der `/agent-flow:reconcile`-Läufe (Stufe 1 Form + Stufe 2 Inhalt). **Jeder** Lauf schreibt **genau einen** Block, auch ohne jede Änderung (No-Op) — so bleibt „gelaufen, nichts nötig" von „nie gelaufen" unterscheidbar. Kopf = Datum, darunter bei Änderungen **je eine Zeile pro berührtem Dokument** (z.B. „Spec X auf use-case-2.0 konvertiert" / „Konzept Y nachgezogen"), bei einem Lauf ohne jede Änderung **genau eine** kanonische „keine Änderung nötig"-Zeile. **Neueste Block oben.** Keine Tabelle, keine Begründung, keine Fundstellen — nur die getroffenen Änderungen bzw. die No-Op-Zeile (durable Historie). Eine einzelne Stufe ohne Änderung trägt keine eigene Zeile bei — die No-Op-Zeile entsteht nur, wenn **weder** Stufe 1 **noch** Stufe 2 etwas geändert haben. Vertrag: `docs/architecture/reconcile-subsystem.md` §4; Acceptance-Kriterien: `docs/specs/reconcile.md` AC10/AC11/AC12.
>
> Geschrieben **ausschließlich** von `scripts/spec-audit-append.sh` (append-prepend — neuer Block wird unter diesem Kopf eingefügt, ältere Blöcke rutschen nach unten) — mit Dokument-Zeilen bei Änderungen, oder im expliziten `--no-op`-Modus bei einem Lauf ohne jede Änderung (AC12). Diese Datei wird beim ersten Lauf angelegt (egal ob mit Änderungen oder als No-Op) — von Hand nichts darunter ergänzen.
