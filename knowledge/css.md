- `css/R05` вҖ” Light/Dark-Farben mit `light-dark(<light>, <dark>)` in **einer** Deklaration statt doppelter `prefers-color-scheme`-BlГ¶cke. **Voraussetzung:** `color-scheme: light dark` (i. d. Quelle: [MDN вҖ” `:has()`](https://developer.mozilla.org/en-US/docs/Web/CSS/:has).
- `css/R07` вҖ” Natives CSS Nesting (ohne Build-Step, ohne Sass) ist Baseline вҖһNewly available" seit **2023-12-11**, erwartet вҖһWidely available" ab **2026-06-11**. Wichtigste Falle: `&` ist bei Pseudo-Klassen/Pseudo-Elementen und Compound-Selektoren **zwingend** (`.card { &:hover {} }` korrekt); bei Nachfahren-Selektoren optional (`.card { .title {} }` = `.card .title`). Kein String-Concatenation Г  la Sass (`.block { &__element {} }` ist **ungГјltig** вҖ” erzeugt keinen Typ-Selektor). Quelle: [MDN вҖ” CSS nesting](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_nesting) В· [web-features-explorer вҖ” nesting](https://web-platform-dx.github.io/web-features-explorer/features/nesting/).
- `css/R08` вҖ” `color-mix(in <colorspace>, <color1> [pct], <color2> [pct])` mischt zwei Farben in einem gewГӨhlten Farbraum (z. Ersetzt preprocessor-Tints/-Shades durch reines CSS (`color-mix(in oklab, var(--color-primary) 80%, white)`). Quelle: [MDN вҖ” `color-mix()`](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value/color-mix) В· [web-features-explorer вҖ” color-mix](https://web-platform-dx.github.io/web-features-explorer/features/color-mix/).
- `css/R09` вҖ” `@starting-style` definiert den **Ausgangs-Zustand** fГјr CSS-Transitions beim ersten Render eines Elements (Entry-Transition) oder wenn `display` von `none` wechselt. Ohne `@starting-style` werden Transitions beim ersten Paint **nicht ausgelГ¶st**. Baseline вҖһNewly available" seit **2024-08-06** (~86 % globale Abdeckung); als Progressive Enhancement einsetzbar (ГӨltere Browser Гјberspringen die Eintritts-Animation, Layout bleibt korrekt). Die Falle ist reine **Source-Order**: `@starting-style` hat dieselbe SpezifitГӨt wie das Haupt-Ruleset, weshalb es im Stylesheet **danach** stehen muss вҖ” steht es davor, gewinnt das Haupt-Ruleset per Source-Order und die Entry-Transition bleibt aus. Quelle: [MDN вҖ” `@starting-style`](https://developer.mozilla.org/en-US/docs/Web/CSS/@starting-style) В· [CSS Transitions Level 2 В§3.3](https://drafts.csswg.org/css-transitions-2/#defining-before-change-style) В· [web-features-explorer вҖ” starting-style](https://web-platform-dx.github.io/web-features-explorer/features/starting-style/).
- `css/R10` вҖ” **View Transitions** (same-document): `document.startViewTransition(() => updateDOM())` + `view-transition-name: <ident>` auf beteiligten Elementen ermГ¶glicht gerenderte ГңbergГӨnge ohne manuelles Snapshot-Handling. Baseline **вҖһNewly available" seit 2025-10-14** (Chrome 111+, Edge 111+, Firefox 144+, Safari 18+). Zwei harte Fallen: (1) **Doppelter Name** вҖ” haben zwei gleichzeitig sichtbare Elemente denselben `view-transition-name`, schlГӨgt `ViewTransition.ready` fehl und der Гңbergang wird **still Гјbersprungen**; (2) **Reduced Motion** вҖ” View Transitions sind Animationen und mГјssen mit `@media (prefers-reduced-motion: reduce) { ::view-transition-old(*), ::view-transition-new(*) { animation: none; } }` deaktiviert werden (R03 gilt hier analog). Quelle: [MDN вҖ” View Transition API](https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API) В· [web-features-explorer вҖ” view-transitions](https://web-platform-dx.github.io/web-features-explorer/features/view-transitions/) В· [web.dev вҖ” same-document view transitions Baseline](https://web.dev/blog/same-document-view-transitions-are-now-baseline-newly-available).
- `css/R11` вҖ” `text-wrap: balance` (oder `text-wrap-style: balance`) fГјr **kurze Mehrzeiler** (Headings, Captions, Blockquotes) eliminiert einzelne ГјberhГӨngende WГ¶rter am Zeilenende, indem Zeilen gleichmГӨssig aufgeteilt werden. Baseline **вҖһNewly available" seit 2024-05-13**. Quelle: [MDN вҖ” `text-wrap`](https://developer.mozilla.org/en-US/docs/Web/CSS/text-wrap) В· [MDN вҖ” `text-wrap-style`](https://developer.mozilla.org/en-US/docs/Web/CSS/text-wrap-style) В· [web-features-explorer вҖ” text-wrap-balance](https://web-platform-dx.github.io/web-features-explorer/features/text-wrap-balance/).
- `css/R12` вҖ” **CSS Subgrid** (`grid-template-columns: subgrid` / `grid-template-rows: subgrid`) lГӨsst einen verschachtelten Grid-Container die Track-GrГ¶ssen des Eltern-Grids erben, statt eigene zu definieren вҖ” ermГ¶glicht tiefe Ausrichtung ohne fixed-Werte oder JS. Baseline **вҖһWidely available" seit 2026-03-15** вҖ” produktionsreif ohne Polyfill. Harte Falle: Wenn **beide** Dimensionen auf `subgrid` gesetzt sind, werden **keine impliziten Tracks** erzeugt; ГјberschГјssige Items stapeln sich im letzten Track. Quelle: [MDN вҖ” Subgrid](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_grid_layout/Subgrid) В· [web-features-explorer вҖ” subgrid](https://web-platform-dx.github.io/web-features-explorer/features/subgrid/).
- `css/R13` вҖ” **CSS Anchor Positioning**: `anchor-name: --x` auf einem beliebigen Element + `position-anchor: --x` auf einem `position: absolute`/`fixed`-Element + `anchor()`/`anchor-size()` in dessen Inset-/Size-Properties positionieren/skalieren dieses Element relativ zum Anker вҖ” ersetzt JS-Tooltip-/Popover-Positionierungs-Libraries. Baseline **вҖһNewly available" seit Januar 2026**. Drei harte Fallen: (1) `anchor()` **erfordert** einen Fallback-Parameter (`anchor(right, 10px)`) вҖ” ohne Fallback ist die Deklaration ungГјltig und wird verworfen; (2) mehrere Anker mit **demselben** `anchor-name` ohne `anchor-scope` binden ALLE positionierten Elemente an den **letzten** Anker in Source-Order, nicht den intendierten; (3) wird der Anker `display: none` gesetzt, bricht die Bindung вҖ” das positionierte Element fГӨllt auf normales `absolute`/`fixed` gegen seinen nГӨchsten positionierten Vorfahren zurГјck (Progressive Enhancement via `@supports (anchor-name: --test)` einsetzen). Quelle: [MDN вҖ” CSS anchor positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning) В· [MDN вҖ” Using CSS anchor positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning/Using).

- `css/R15` вҖ” **Container Style Queries** (`@container style(--custom-prop)`, `@container style(--custom-prop: wert)`) erweitern `css/R04` um Styling anhand einer **Custom Property** eines Vorfahren-Containers statt dessen GrГ¶sse вҖ” anders als Size Queries wird dafГјr **kein** `container-type` benГ¶tigt. Zusatz-Falle: bei registrierten Custom Properties (`@property`) prГјft `style(--var)` ohne Wert gegen deren `initial-value`; bei unregistrierten matcht `style(--var: wert)` rein token-basiert, nicht semantisch-ГӨquivalent (`blue` вү  `#0000ff` ohne `@property`-`syntax`). Quelle: [MDN вҖ” Container size and style queries](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_containment/Container_size_and_style_queries).

## Reviewer-Checklist
- Werte auГҹerhalb der Spacing-/Token-Skala вҶ’ **Important**.
- Kontrast < WCAG (Body 4.5:1 / Large 3:1) вҖ” **berechnen**, nicht schГӨtzen вҶ’ **Critical**.
- Animationen ohne `prefers-reduced-motion`-BerГјcksichtigung вҶ’ **Important**.
- `@container`-Query ohne `container-type`/`container` auf einem Vorfahren вҶ’ Query ist wirkungslos вҶ’ **Important** (`css/R04`).
- `light-dark()` verwendet, aber kein `color-scheme: light dark` gesetzt вҶ’ Funktion greift nicht вҶ’ **Important** (`css/R05`).
- `:has()` auГҹerhalb von `:is()`/`:where()` in einem Selektor, der auch unsupporteten Browsern dienen muss вҶ’ kompletter Block-Ausfall вҶ’ **Important** (`css/R06`).
- Sass-style `&__element`-Concatenation in nativem CSS вҶ’ ungГјltig (kein Type-Selektor-Prefix) вҶ’ **Important** (`css/R07`).
- `color-mix()` mit `srgb` statt `oklab`/`oklch` fГјr Tints/Shades вҶ’ perceptuell ungleichmГӨssige Ergebnisse вҖ” `oklab` bevorzugen вҶ’ **Minor** (`css/R08`).
- `@starting-style`-Block vor dem Haupt-Ruleset вҶ’ gleiche SpezifitГӨt, Haupt-Styles gewinnen per **Source-Order** (`@starting-style` ist separater before-change-state-Mechanismus, kein SpezifitГӨts-Konkurrent), Entry-Transition fehlt вҶ’ **Important** (`css/R09`).
- `view-transition-name` auf zwei gleichzeitig sichtbaren Elementen mit demselben Wert вҶ’ `ViewTransition.ready` schlГӨgt fehl, Гңbergang wird still Гјbersprungen вҶ’ **Critical** (`css/R10`).
- View-Transition-Animationen ohne `prefers-reduced-motion`-Guard (`::view-transition-old(*)/::view-transition-new(*)`) вҶ’ **Important** (`css/R10` + `css/R03`).
- `text-wrap-style: pretty` in Produktion ohne Firefox-Fallback вҶ’ kein Baseline-Status (Firefox fehlt, Stand 2026-06); Chromium 130+ und Safari 26 unterstГјtzen es, Firefox nicht вҶ’ **Important** (`css/R11`).
- Subgrid (`grid-template-columns/rows: subgrid`) auf beiden Achsen ohne `grid-auto-rows` вҶ’ ГјberschГјssige Items stapeln sich im letzten Track, kein implizites Wachstum вҶ’ **Important** (`css/R12`).
- `anchor()`/`anchor-size()` ohne Fallback-Parameter вҶ’ ungГјltige Deklaration, wird verworfen вҶ’ **Important** (`css/R13`).
- Mehrere Elemente mit demselben `anchor-name` ohne `anchor-scope` вҶ’ alle positionierten Elemente binden fГӨlschlich an den letzten Anker in Source-Order вҶ’ **Important** (`css/R13`).
- `contrast-color()`-Resultat bei erkennbaren Mid-Tone-Basisfarben ohne rechnerische Kontrast-NachprГјfung Гјbernommen вҶ’ potenziell < 4,5:1 вҶ’ **Important** (`css/R14`).
- `@container style(...)` mit einer Standard-CSS-Property statt einer Custom Property (z. B. `style(font-weight: bold)`) вҶ’ (Stand 2026) nicht implementiert, Query wirkungslos вҶ’ **Important** (`css/R15`).

## Test-Approach
- Visueller Smoke; Kontrast der geГӨnderten Farbpaare berechnet.
- Container-Queries: Komponente isoliert in schmalem/breitem Container rendern (nicht nur Viewport resizen) und Layout-Wechsel verifizieren.
- `light-dark()`: in beiden `color-scheme`-Modi (Light/Dark) prГјfen.
- CSS Nesting: in DevTools CSSOM verifizieren dass Selektoren korrekt aufgelГ¶st werden (Compound-Selektoren mit `&`).
- `@starting-style`: Element aus DOM entfernen und wieder einfГјgen (oder `display: none` вҶ’ sichtbar) вҖ” Entry-Transition muss ausgelГ¶st werden.
- View Transitions: `document.startViewTransition()` auslГ¶sen; in DevTools Animations-Panel prГјfen ob `::view-transition-old/new` erzeugt werden; doppelte `view-transition-name`-Werte вҶ’ Animation fehlt komplett (kein Fehler in der Console ohne explizites Promise-Handling).
- `text-wrap: balance`: in schmalen Containern auf kurze Headings (вүӨ6 Zeilen) anwenden und ZeilenbrГјche visuell verifizieren; in Browser ohne Support (Fallback: normal wrapping) prГјfen dass Layout intakt bleibt.
- Subgrid: Nested-Grid-Items in einem Eltern-Grid rendern und prГјfen, dass Spalten/Zeilen mit dem Parent-Grid ausgerichtet sind (DevTools Grid-Overlay nutzen).
- Anchor Positioning: Anker per `display: none` ausblenden und verifizieren, dass das positionierte Element auf den Fallback (nГӨchster positionierter Vorfahre) zurГјckfГӨllt; mehrere gleichnamige Anker mit und ohne `anchor-scope` testen.
- `contrast-color()`: KontrastverhГӨltnis der zurГјckgegebenen Farbe gegen die Basisfarbe **rechnerisch** prГјfen (nicht nur visuell), insbesondere bei Mid-Tone-Basisfarben.
- Container Style Queries: Custom Property am Container zur Laufzeit ГӨndern und verifizieren, dass die `@container style(...)`-Regel greift вҖ” auch ohne gesetztes `container-type`.
